# Review Cycle — Claude Code Plugin

> [English version](README.md)

Автоматический цикл ревью PR для Claude Code. Запрашивает ревью, интеллектуально сортирует комментарии ревьюеров (ботов и людей), применяет исправления и повторяет до аппрува — затем squash-мержит.

## Установка

```bash
git clone https://github.com/axisrow/claude-code-review-cycle-skill.git
cp -r claude-code-review-cycle-skill/skills/review-cycle ~/.claude/skills/
```

### Требования

- [GitHub CLI](https://cli.github.com/) (`gh`), авторизованный в вашем аккаунте
- Claude Code с ревьюером через GitHub Actions (`claude[bot]`) или любой другой PR-ревьюер

### Настройка

Перед использованием скилла выполните команду один раз:

```
/install-github-app
```

Это устанавливает GitHub App и CI, чтобы Claude Code мог делать ревью и редактировать комментарии на pull requests.

## Использование

```
/cycle-review (cr) [pr-number]
```

Если номер PR не указан, скилл определяет его автоматически из текущей ветки.

## Как это работает

Скилл запускает автоматический цикл из 7 шагов:

### 1. Проверка более ранних PR

Перед началом проверяет, есть ли открытые PR с меньшими номерами. Если найдены — спрашивает: смерджить сначала более ранний, работать параллельно или пропустить проверку.

### 2. Запрос ревью

Оставляет комментарий с просьбой сфокусироваться только на критичных проблемах (баги, безопасность, логические ошибки, потеря данных, производительность). Косметические замечания явно не приветствуются.

### 3. Ожидание ответа ревьюера

Вместо фиксированного таймаута скилл **поллит комментарий ревьюера по ID**:
- Находит последний комментарий от `claude[bot]` через GitHub API
- Опрашивает каждые 30 секунд, проверяя body на маркер `Claude finished`
- Ревью считается завершённым когда чеклист прогресса исчезает и появляется маркер завершения
- Таймаут: 7 минут с момента создания комментария

### 4. Анализ и сортировка комментариев

Собирает комментарии от **всех ревьюеров** (ботов и людей) — как issue-комментарии, так и PR review comments. Субагент сортирует каждый комментарий, читая реальный код:

| Вердикт | Действие |
|---|---|
| `FIX` | Применить исправление (только после проверки утверждения по реальному коду) |
| `ALREADY_FIXED` | Ответить с указанием коммита, где это уже исправлено |
| `SKIP` | Ответить, что это косметическое замечание |
| `IRRELEVANT` | Ответить, что комментарий не относится к PR |
| `CONFLICTING` | Процитировать противоречащий комментарий, попросить уточнение |
| `HALLUCINATION` | Ответить с доказательствами из кодовой базы, опровергающими утверждение |

### 5. Исправление

Применяет фиксы только для вердикта `FIX`. Запускает линтер и тесты перед продолжением.

### 6. Коммит и пуш

Conventional commit, пуш в remote, возврат к шагу 2.

### 7. Завершение

При аппруве без оставшихся комментариев — squash-мерж и удаление ветки.

## Ключевые особенности

- **Умный поллинг** — отслеживает комментарий ревьюера по ID вместо фиксированного ожидания
- **Мульти-ревьюер** — обрабатывает комментарии от всех ревьюеров, не только от бота
- **Проверка ранних PR** — предупреждает об открытых PR с меньшими номерами перед стартом
- **Верификация утверждений** — проверяет каждое утверждение ревьюера по реальной кодовой базе
- **Детекция галлюцинаций** — ловит несуществующие функции, неверные номера строк, ложные конвенции
- **Обработка противоречий** — обнаруживает конфликтующие запросы между циклами ревью
- **Пропуск косметики** — пропускает стилевые замечания с вежливым объяснением
- **Линтер и тесты** — запускает `ruff` и `pytest` перед каждым пушем
